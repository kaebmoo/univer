#### **เกริ่นนำ: ปัญหาความปวดหัวของการตรวจสอบรายงาน**

ทุกเดือน ทีม Finance มักต้องเผชิญกับ "สงครามข้อมูล" เรามีไฟล์จากหลายแหล่ง ทั้ง CSV จากระบบหลัก, Excel รายงานผลการดำเนินงาน (P\&L) ที่แยกย่อยหลาย Sheet และไฟล์ Text งบการเงินจากระบบบัญชี

ความท้าทายคือการต้องนำข้อมูลทั้งหมดนี้มา **"กระทบยอด (Reconciliation)"** เพื่อยืนยันความถูกต้อง:

  * ต้องเปิดไฟล์เทียบทีละบรรทัด
  * ต้องกดเครื่องคิดเลขเช็คสูตรว่า Excel คำนวณถูกไหม
  * ต้องระวังเรื่องการปัดเศษทศนิยม
  * **ใช้เวลาเฉลี่ย:** 3-4 ชั่วโมงต่อเดือน (และเพิ่มขึ้นหากตัวเลขไม่ตรง)
  * **ความเสี่ยง:** Human Error จากความเหนื่อยล้า

-----

#### **The Solution: ผู้ช่วยตรวจสอบอัจฉริยะ**

เพื่อแก้ไขปัญหานี้ ทีมงานได้พัฒนาเครื่องมือ **Automated P\&L Reconciliation Tool (v1.0)** ด้วยภาษา **Python** ซึ่งทำหน้าที่เปรียบเสมือน "ผู้ช่วย Auditor" ที่ทำงานได้ตลอด 24 ชั่วโมง โดยไม่มีความเหนื่อยล้า



**จุดเด่นของเครื่องมือนี้:**

1.  **เร็ว:** ใช้เวลาประมวลผลเพียง **10-30 วินาที** (เทียบกับคนทำ 3 ชม.)
2.  **แม่นยำ:** ตรวจสอบทุกจุดทศนิยม และแปลง Format ตัวเลขไทย/สากลให้อัตโนมัติ
3.  **ครบถ้วน:** รองรับการตรวจทั้งงวดรายเดือน (MTH) และยอดสะสม (YTD) พร้อมกัน

-----

#### **How It Works: หลักการทำงานของระบบ**

โปรแกรมทำงานโดยการอ่านไฟล์ข้อมูลดิบและรายงานพร้อมกัน 3 แหล่ง และทำการตรวจสอบ 3 ระดับ (3-Stage Validation) ดังนี้:

**1. ระดับความครบถ้วน (Completeness Check)**

  * **โจทย์:** ข้อมูลจากระบบหลัก (CSV) ถูกนำไปทำรายงานครบหรือไม่?
  * **การทำงาน:** โปรแกรมจะ Sum ยอดรวมจาก Source File เทียบกับยอดรวมใน Excel Report ทันที หากหายไปแม้แต่บาทเดียว ระบบจะแจ้งเตือน

**2. ระดับความสอดคล้อง (Consistency Check)**

  * **โจทย์:** รายงานมิติต่าง ๆ (ต้นทุน vs บัญชี) ตรงกันหรือไม่?
  * **การทำงาน:** ระบบจะ Cross-check ข้าม Sheet ระหว่างรายงาน "ประเภทต้นทุน" และ "หมวดบัญชี" ในทุก Business Unit เพื่อให้มั่นใจว่ามองมุมไหนตัวเลขก็ต้องเท่ากัน
  * **Special Feature:** มีฟังก์ชัน **Internal Math Check** ที่คอยตรวจสูตรใน Excel ว่า `รายได้ - ค่าใช้จ่าย - ภาษี = กำไรสุทธิ` จริงหรือไม่ (ป้องกันสูตร Excel เพี้ยน)

**3. ระดับความถูกต้องบัญชี (Financial Tie-out)**

  * **โจทย์:** ตัวเลขสุดท้ายตรงกับงบการเงินหรือไม่?
  * **การทำงาน:** ระบบจะดึงตัวเลขบรรทัดสุดท้ายจาก Excel ไปเทียบกับไฟล์ Text (งบการเงินจริงจากระบบ) เป็นด่านสุดท้าย

-----


#### **วิธีใช้งาน (สำหรับ User)**

เราออกแบบมาให้ใช้งานง่ายและยืดหยุ่นกว่าเดิม **โดยไม่ต้องเปิดแก้ไขโค้ดแม้แต่บรรทัดเดียว**:

1.  **เตรียมไฟล์:** วางไฟล์ข้อมูล (Excel, CSV, Text) ทั้งหมดไว้ในโฟลเดอร์เดียวกับโปรแกรม

2.  **รันโปรแกรม:** เปิด Command Line (หรือ Terminal) ในโฟลเดอร์นั้น แล้วพิมพ์คำสั่งระบุวันที่ที่ต้องการตรวจ:

    ```bash
    python pl_reconciliation_enhanced.py --date 20251031
    ```

    *(เพียงเปลี่ยนตัวเลขหลัง `--date` เป็นวันสิ้นเดือนที่ต้องการ ระบบจะไปดึงไฟล์ MTH และ YTD ของเดือนนั้นมาตรวจสอบให้อัตโนมัติ)*

3.  **เลือกโหมด (Optional):** หากต้องการเปลี่ยนความละเอียดในการตรวจ สามารถเพิ่มคำสั่งต่อท้ายได้:

      * `--mode basic` (ตรวจเร็ว ดูแค่ Net Profit)
      * `--mode enhanced` (มาตรฐาน - ตรวจครบทุกมิติ **แนะนำ**)
      * `--mode comprehensive` (เจาะลึก)

4.  **รอผลลัพธ์:** ภายในไม่กี่วินาที หน้าจอจะแสดงสถานะ **✅ PASS** หรือ **❌ FAIL** พร้อมสรุปยอด Diff ในแต่ละ Sheet ทันที
